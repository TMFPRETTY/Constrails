# Security Policy

## Reporting a vulnerability

If you discover a security issue in Constrails, please report it privately rather than opening a public issue with exploitation details.

Recommended approach:
- contact the maintainer directly through a private channel if one is available
- or open a minimal GitHub issue asking for a private reporting path without disclosing sensitive details

Please include:
- affected component(s)
- reproduction steps
- impact assessment
- any suggested mitigations

## Scope

Because Constrails is an Agent Safety System, security issues involving policy bypass, approval bypass, audit tampering, sandbox escape, capability escalation, or unsafe tool execution paths should be treated as high priority.

## Expectations

This project is currently early alpha software. Security hardening is in progress, and responsible disclosure is appreciated.
